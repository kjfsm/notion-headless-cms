# v1.0 移行ガイド

## 要約

- コレクション・グローバルのメソッド名が整理され、`$` プレフィックスが廃止された
- キャッシュ設定オプションの構造が変わった（`ttlMs` のネスト化・`cache` の配列必須化）
- `nhc.config.ts` の `columnMappings` が `fieldMappings` に改名された
- `GetOptions` / `RevalidateResult` 型が改名された
- 新しいエラーマッチング API (`matchCMSError` / `err.is()` / `err.inNamespace()`) が追加された
- `adapter-next` に `createNextHandler` が追加され、個別ハンドラ関数は非推奨になった

影響を受けるパッケージ: `@notion-headless-cms/core` / `@notion-headless-cms/adapter-next` / CLI (`nhc.config.ts`)

---

## 破壊的変更一覧

| 変更区分 | 変更前 | 変更後 |
|---|---|---|
| メソッド | `cms.posts.get(slug)` | `cms.posts.find(slug)` |
| 戻り値 | `find` が非 null を返す | `find` は `ItemWithContent<T> \| null` を返す |
| メソッド | `cms.posts.slugs()` | `cms.posts.params()` |
| メソッド | `cms.posts.revalidate(slug, ver)` | `cms.posts.check(slug, ver)` |
| プロパティ | `cms.$collections` | `cms.collections` |
| メソッド | `cms.$invalidate()` | `cms.invalidate()` |
| メソッド | `cms.$handler(opts)` | `cms.handler(opts)` |
| メソッド | `cms.$getCachedImage(hash)` | `cms.getCachedImage(hash)` |
| オプション | `cache: adapter`（単体） | `cache: [adapter]`（常に配列） |
| オプション | `ttlMs: number`（トップレベル） | `swr: { ttlMs: number }`（ネスト） |
| 設定ファイル | `columnMappings` in `nhc.config.ts` | `fieldMappings` |
| 型 | `GetOptions` | `FindOptions` |
| 型 | `RevalidateResult` | `CheckResult` |
| adapter-next | `createImageRouteHandler(cms)` | `createNextHandler(cms, opts?)` を使うこと（非推奨） |
| adapter-next | `createCollectionRevalidateRouteHandler(cms, opts)` | `createNextHandler(cms, opts?)` を使うこと（非推奨） |
| adapter-next | `createInvalidateAllRouteHandler(cms, opts)` | `createNextHandler(cms, opts?)` を使うこと（非推奨） |

## Before / After コード例

```ts
// ===== Before (v0.x) =====

import { createCMS } from "./generated/nhc";
import { memoryCache } from "@notion-headless-cms/cache";

const cms = createCMS({
  notionToken: process.env.NOTION_TOKEN!,
  cache: memoryCache(),        // 単体オブジェクト
  ttlMs: 5 * 60_000,          // トップレベル
});

// コレクション操作
const slugs = await cms.posts.slugs();          // generateStaticParams 用
const post = await cms.posts.get("my-post");    // 非 null 保証なし
const result = await cms.posts.revalidate("my-post", post.updatedAt);

// グローバル操作
console.log(cms.$collections);
await cms.$invalidate();
const img = await cms.$getCachedImage(hash);
const handler = cms.$handler({ webhookSecret: "..." });

// nhc.config.ts
export default defineConfig({
  columnMappings: { title: "タイトル" },
});

// adapter-next (Next.js)
import { createImageRouteHandler, createCollectionRevalidateRouteHandler } from "@notion-headless-cms/adapter-next";
export const GET = createImageRouteHandler(cms);
export const POST = createCollectionRevalidateRouteHandler(cms, { secret: "..." });

// 型
import type { GetOptions, RevalidateResult } from "@notion-headless-cms/core";
```

```ts
// ===== After (v1.0) =====

import { createCMS } from "./generated/nhc";
import { memoryCache } from "@notion-headless-cms/cache";

const cms = createCMS({
  notionToken: process.env.NOTION_TOKEN!,
  cache: [memoryCache()],          // 常に配列
  swr: { ttlMs: 5 * 60_000 },     // swr ネスト
});

// コレクション操作
const params = await cms.posts.params();          // generateStaticParams 用
const post = await cms.posts.find("my-post");     // null を返す可能性あり
if (post) {
  const result = await cms.posts.check("my-post", post.updatedAt);
}

// グローバル操作
console.log(cms.collections);
await cms.invalidate();
const img = await cms.getCachedImage(hash);
const handler = cms.handler({ webhookSecret: "..." });

// nhc.config.ts
export default defineConfig({
  fieldMappings: { title: "タイトル" },
});

// adapter-next (Next.js) — 統合ハンドラ
import { createNextHandler } from "@notion-headless-cms/adapter-next";
const handler = createNextHandler(cms, { webhookSecret: "..." });
export const GET  = handler;
export const POST = handler;

// 型
import type { FindOptions, CheckResult } from "@notion-headless-cms/core";
```

## 新規追加 API

### `matchCMSError` / `err.is()` / `err.inNamespace()`

エラーコードに基づくエルゴノミックなパターンマッチングが可能になった。

```ts
import { matchCMSError, CMSError } from "@notion-headless-cms/core";

// matchCMSError でエラーコード別にハンドラを定義
try {
  await cms.posts.find(slug);
} catch (err) {
  matchCMSError(err, {
    "source/fetch_item_failed": (e) => console.error("Notion 取得失敗:", e.message),
    "cache/io_failed": (e) => console.error("キャッシュ失敗:", e.message),
    _: (e) => { throw e; },
  });
}

// インスタンスメソッドで判定
if (err instanceof CMSError) {
  err.is("cache/io_failed");      // 完全一致
  err.inNamespace("source/");     // 名前空間前方一致
}
```

### `createNextHandler` (adapter-next)

画像プロキシと Webhook 受信を 1 つのルートハンドラにまとめた統合 API。

```ts
// app/api/cms/[...route]/route.ts
import { createNextHandler } from "@notion-headless-cms/adapter-next";
const handler = createNextHandler(cms, { webhookSecret: process.env.REVALIDATE_SECRET });
export const GET  = handler;
export const POST = handler;
```

## 自動置換スクリプト（参考）

単純な識別子の置換は `sed` で対応できる。適用前に差分を確認すること。

```bash
# メソッド名の置換
find . -name "*.ts" -not -path "*/node_modules/*" \
  -exec sed -i \
    -e 's/cms\.posts\.get(/cms.posts.find(/g' \
    -e 's/cms\.posts\.slugs(/cms.posts.params(/g' \
    -e 's/cms\.posts\.revalidate(/cms.posts.check(/g' \
    -e 's/cms\.\$collections/cms.collections/g' \
    -e 's/cms\.\$invalidate(/cms.invalidate(/g' \
    -e 's/cms\.\$handler(/cms.handler(/g' \
    -e 's/cms\.\$getCachedImage(/cms.getCachedImage(/g' \
    -e 's/columnMappings/fieldMappings/g' \
    -e 's/GetOptions/FindOptions/g' \
    -e 's/RevalidateResult/CheckResult/g' \
  {} +
```

`ttlMs` のネスト化と `cache` の配列化は構造変更のため、手動で対応すること。
